## 功能 1：AI 销售助手仪表盘 (AiSalesAssistantDashboard)

**页面路径**：`/ai-sales`

**功能描述**：
AI 销售助手的首页仪表盘，提供全局视图。启动时并行加载风险预警摘要和待处理销售建议，以卡片和列表形式呈现。

**四个标签页**：

| 标签 | 功能 |
|------|------|
| 概览 | 显示风险预警统计卡片（总数/严重/高/中/低/已解决）+ Top 5 待处理建议 |
| 知识库 | 知识库快速搜索——输入问题，AI 从企业知识库中检索并回答，附带引用来源 |
| 建议 | 展示所有待处理的 AI 销售建议列表 |
| 风险预警 | 展示所有风险预警列表，支持状态筛选和操作 |

**数据流**：
1. `useEffect` → 并行调用 `aiRiskAlertsClient.getSummary()` + `aiSalesSuggestionsClient.list()`
2. 概览页渲染 `StatCard` × 6 + `SuggestionCard` 列表
3. 知识库搜索：用户输入 → `aiKnowledgeBaseClient.query()` → 显示 AI 回答 + 来源引用
4. 建议/预警标签页各自独立加载数据

## 功能 2：AI 客户画像 (AiCustomerProfileCard)

**使用位置**：嵌入在 Company/Person 详情页中

**功能描述**：
为指定的公司或联系人自动生成 360 度 AI 客户画像。包括：

- **情感分析分数** (sentimentScore)：-1 到 1，反映客户对公司的整体情感倾向
- **流失风险分数** (churnRiskScore)：0 到 1，高分表示客户可能流失
- **增销潜力分数** (upsellPotentialScore)：0 到 1，高分表示有增销/交叉销售机会
- **参与度等级** (engagementLevel)：高/中/低/无
- **标签** (tags)：AI 自动打标的客户特征标签
- **关键洞察** (insights)：分行为/财务/沟通/风险/机会五类，附置信度百分比
- **生成时间戳** + 刷新按钮

**数据流**：
1. 组件挂载 → `aiCustomerProfileClient.getByCompany()` 尝试加载已有画像
2. 若无画像 → 显示"生成画像"按钮
3. 点击生成 → `aiCustomerProfileClient.generate()` → 显示完整画像
4. 点击刷新 → 重新生成最新画像

## 功能 3：AI 销售建议 (AiSalesSuggestionsList)

**使用位置**：嵌入在 Company/Opportunity/Person 详情页中，也可在仪表盘中查看

**功能描述**：
针对特定目标（公司/商机/联系人）生成 AI 驱动的销售行动建议。每条建议包含：

- **建议类型** (suggestionType)：下一步行动/交叉销售/增销/跟进/会议准备
- **优先级** (priority)：高/中/低
- **标题 + 描述**：AI 生成的建议内容
- **理由** (rationale)：AI 解释为什么提这个建议
- **建议操作** (suggestedActions)：具体可执行的操作步骤
- **状态流转**：待处理 → 已接受 → 已完成（或 → 已忽略）
- **反馈机制** (feedbacks)：用户可对建议打分和标记是否有帮助

**状态筛选按钮**：待处理 / 已接受 / 已完成 / 已忽略

**数据流**：
1. 组件挂载 → `aiSalesSuggestionsClient.list()` 加载已有建议
2. 点击"生成新建议" → `aiSalesSuggestionsClient.generate()` → 刷新列表
3. 点击接受/完成/忽略 → `aiSalesSuggestionsClient.updateStatus()` → 刷新列表

## 功能 4：文档问答 (AiDocumentQAChat)

**使用位置**：嵌入在仪表盘"知识库"标签或独立使用

**功能描述**：
两种模式的 AI 对话界面：

### 模式 A：知识库模式 (knowledge-base)
- 直接向企业知识库提问，AI 搜索相关文档并回答
- 每次回答附带来源引用（文档标题 + 摘要摘录）

### 模式 B：文档问答模式 (document-qa)
- 左侧边栏：上传文档（粘贴文本 + 标题）→ 文档列表
- 点击文档创建对话 → 基于选中文档进行问答
- 支持对话历史：列出所有对话，可切换查看
- 每条 AI 回复包含引文（chunk 级别）

**通用交互**：
- 聊天界面（用户消息右对齐蓝色、AI 消息左对齐灰色）
- 输入框 + 发送按钮，支持 Enter 发送
- 消息列表自动滚动到底部
- 加载中显示 "..."
- 错误处理：AI 服务异常时显示友好错误消息

## 功能 5：风险预警 (AiRiskAlertList)

**页面路径**：`/ai-sales/risk-alerts`，也嵌入在仪表盘中

**功能描述**：
基于规则引擎的 AI 风险监控系统。自动检测 CRM 中的异常并生成预警。

**预警规则类型**：
| 规则类型 | 说明 |
|---------|------|
| payment_overdue | 付款逾期检测 |
| opportunity_stagnant | 商机停滞检测 |
| deal_slipping | 交易下滑检测 |
| customer_churn | 客户流失风险 |
| custom | 自定义规则 |

**预警严重级别**：严重(critical) / 高(high) / 中(medium) / 低(low)

**预警状态流转**：待处理(open) → 已确认(acknowledged) → 已解决(resolved)（或 → 已忽略(dismissed)）

**每条预警包含**：
- 标题 + 描述
- AI 分析内容（aiAnalysis）
- 建议操作（suggestedActions）
- 时间线（创建时间、确认时间、解决时间）
- 关联的目标对象（公司/商机/联系人）

**顶部摘要栏**：待处理总数 / 严重 / 高 / 中 / 低 / 7天内已解决

**数据流**：
1. 加载 → `aiRiskAlertsClient.getSummary()` + `aiRiskAlertsClient.listAlerts()` 并行
2. 状态筛选 → 重新 `listAlerts({ status })`
3. 确认/解决/忽略 → `aiRiskAlertsClient.updateAlertStatus()` → 刷新

## 功能 6：AI 对话 (AiChatPage)

**页面路径**：`/ai-sales/chat`

**功能描述**：
独立的 AI 对话页面，与 AI 销售助手进行自由对话。

- 左侧对话列表：历史对话，显示标题 + 消息数
- 点击"新建对话"清空上下文
- 支持对话持久化（后端保存 conversationId）
- 聊天界面：用户消息右对齐蓝色、AI 消息左对齐灰色
- Thinking... 加载指示器
- 错误处理：捕获异常并显示错误消息

**数据流**：
1. 挂载 → 加载历史对话列表
2. 发送消息 → POST `/chat` → AI 回复 → 自动滚动
3. 切换历史对话 → 加载对话消息

## 功能 7：知识库管理 (AiKnowledgeBasePage)

**页面路径**：`/ai-sales/knowledge-base`

**功能描述**：
知识库文档管理 + 文档问答。

- **文件上传区**：点击或拖放上传 PDF/DOCX/PPTX/XLSX/TXT/MD/CSV
- **文档列表**：显示所有已上传文档（标题 + 类型标签）
- **文档删除**：🗑 按钮删除文档
- **知识库问答**：右侧聊天区，基于所有上传文档进行问答
- 答案附带来源引用（📎 Sources: [1] 文档标题）
- 搜索中显示"正在搜索知识库..."
